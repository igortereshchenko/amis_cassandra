Пакет операцій - поєднання операцій для досягнення їх атомарності - виконання або всіх операцій, або жодної.
Особливості пакету операцій:
- обмеженість по часу (задається адміністратором для вузла)
- кожна операція в пакеті повинна містити умову where
- операції виконуються в довільному порядку
- відповідно всі батчеві операції повинні бути незалежні від порядку їх виконання
- батчі дозволяють робити rollback
Використання батчевих операцій для одного та багатьох партиційних ключів (операції виконуються на одному або декількох вузлах) відрізняється.
Для однієї партиційного ключа:ізольованість (блокування рядку), roll back, restore
Для багатьох партиційних ключів батчеві операції слід використовувати для збереження історії (зберегли і забули) і іноді для зв’язків 1-n. Відсутня ізольованість!
Батчеві операції варто використовувати:
- для insert, update, delete для одного партиційного кліча коли атомарність і ізольованість є обов’язковою. Для однієї партиції кількість задіяних вузлів буде обмежена кількістю реплік.
- щоб переконатись в атомарності при малих insert або update для декількох партицій коли неконсистентність не можливо виникнути.
Батчеві операції не варто використовувати:
- для insert або update декількох ключів, особливо якщо велика кількість ключів задіяна - це сильно впливає на продуктивність.
